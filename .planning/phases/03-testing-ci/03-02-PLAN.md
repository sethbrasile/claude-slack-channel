  <name>Task 1: Create CI workflow (CICD-01)</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create the `.github/workflows/` directory and write `ci.yml`.

The workflow must:
- Trigger on: push to any branch (`branches: ["*"]`) AND pull_request (no branch filter)
- Run a single job named `ci` on `ubuntu-latest`
- Steps in order:
  1. `actions/checkout@v4`
  2. `oven-sh/setup-bun@v2`
  3. `bun install --frozen-lockfile` (never plain `bun install` — lockfile must not change in CI)
  4. `bunx tsc --noEmit` (fail fast before running tests)
  5. `bunx biome check .`
  6. `bun test --coverage` (coverage reported to CI stdout — no threshold enforcement needed)
- No separate jobs for each check — one job with sequential steps per research recommendations

Anti-patterns to avoid:
- Do NOT use `bun install` without `--frozen-lockfile`
- Do NOT split into three separate jobs
- Do NOT add coverage threshold configuration (CICD-01 requires coverage *reported*, not gated)
- Do NOT add any publish steps to the CI workflow

```yaml
name: CI

on:
  push:
    branches: ["*"]
  pull_request:

jobs:
  ci:
    name: Typecheck, Lint, Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install --frozen-lockfile
      - run: bunx tsc --noEmit
      - run: bunx biome check .
      - run: bun test --coverage
```

Write this exact content to `.github/workflows/ci.yml`. Ensure the directory is created.
  </action>

<task type="auto">
  <name>Task 2: Create release workflow (CICD-02, CICD-03)</name>
  <files>.github/workflows/release.yml</files>
  <action>
Write `.github/workflows/release.yml`.

The workflow must:
- Trigger on: push of tags matching `v*` only
- Run a single job named `release` on `ubuntu-latest`
- Declare explicit permissions: `contents: write` (required for gh release create) and `id-token: write` (required for npm provenance)
- Steps in order:
  1. `actions/checkout@v4`
  2. `oven-sh/setup-bun@v2`
  3. `actions/setup-node@v4` with `node-version: "20.x"` and `registry-url: "https://registry.npmjs.org"` — REQUIRED to configure npm auth; skipping this causes "npm ERR! need auth"
  4. `bun install --frozen-lockfile`
  5. `bunx tsc --noEmit` (gate: don't publish broken code)
  6. `bun test` (gate: don't publish failing tests)
  7. `npm publish --provenance --access public` with env `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`
  8. `gh release create "${{ github.ref_name }}" --generate-notes` with env `GH_TOKEN: ${{ github.token }}`

Anti-patterns to avoid:
- Do NOT use `bun publish` — it does not support `--provenance` flag
- Do NOT skip `actions/setup-node@v4` with `registry-url` — npm auth will fail without it
- Do NOT omit `contents: write` permission — gh release create will fail with "Resource not accessible by integration"
- Do NOT omit `id-token: write` permission — provenance attestation requires OIDC token

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

jobs:
  release:
    name: Publish to npm + Create Release
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - uses: actions/setup-node@v4
        with:
          node-version: "20.x"
          registry-url: "https://registry.npmjs.org"
      - run: bun install --frozen-lockfile
      - run: bunx tsc --noEmit
      - run: bun test
      - run: npm publish --provenance --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
      - name: Create GitHub Release
        run: gh release create "${{ github.ref_name }}" --generate-notes
        env:
          GH_TOKEN: ${{ github.token }}
```

Write this exact content to `.github/workflows/release.yml`.
  </action>
  <verify>
    <automated>test -f .github/workflows/release.yml && grep -q "npm publish --provenance" .github/workflows/release.yml && grep -q "gh release create" .github/workflows/release.yml && grep -q "id-token: write" .github/workflows/release.yml && echo "release.yml valid"</automated>
  </verify>
  <done>.github/workflows/release.yml exists with v* tag trigger, contents: write + id-token: write permissions, npm publish --provenance step, and gh release create --generate-notes step.</done>
