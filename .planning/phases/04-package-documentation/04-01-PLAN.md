**slack-app-manifest.yaml** (DOCS-02):
Write a Slack app manifest YAML using manifest schema version 2. Required fields per research:
- `_metadata`: major_version 2, minor_version 1
- `display_information`: name "Claude Slack Channel", description "MCP bridge for Claude Code automation pipelines", background_color "#1A1D21"
- `features.bot_user`: display_name "Claude", always_online true (prevents confusing "offline" status)
- `oauth_config.scopes.bot`: chat:write, channels:history, groups:history (groups:history is required for private channels — omitting causes silent failure)
- `settings.event_subscriptions.bot_events`: message.channels, message.groups
- `settings`: socket_mode_enabled true, is_hosted false, token_rotation_enabled false

Add a comment at the top explaining the app-level token (xapp-) needs `connections:write` scope — this is set separately when enabling Socket Mode in the Slack app settings, not in the manifest. This prevents operator confusion.

DO NOT include scopes the server doesn't use (users:read, etc.). Keep it minimal.

**CRITICAL PITFALLS from research:**
- Include both channels:history AND groups:history (private channel support)
- Include always_online: true in features.bot_user (prevents "offline" confusion)
- Do NOT claim npx works anywhere

**.env.example** (DOCS-03):

- SLACK_BOT_TOKEN — required, starts with xoxb-, comment explaining where to find it
- SLACK_APP_TOKEN — required, starts with xapp-, comment explaining Socket Mode requirement
- SLACK_CHANNEL_ID — required, format C0XXXXXXXXX, comment explaining how to find it (right-click channel > Copy link, extract ID)
