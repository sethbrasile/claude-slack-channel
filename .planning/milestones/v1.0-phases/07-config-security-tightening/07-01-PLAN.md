# Plan 07-01: Config & Security Tightening

## Goal

Apply all five deep-review security findings (M4, M5, M6, M8, L9) as surgical edits across four source files, then extend tests to verify the new behaviors.

## Context

All changes are independent of each other. They touch different functions in different files and can be applied in any order. No new dependencies. No new files (only additions to existing test files).

**Critical project rule:** stdout is sacred — all logging uses `console.error()`. Do NOT touch any stdout-facing code.

## Requirements Addressed

- M4: SLACK_CHANNEL_ID regex validation
- M5: safeErrorMessage in createStderrLogger error method
- M6: Pre-built PERMISSION_ID_RE constant exported from permission.ts
- M8: Simplify double casts in server.ts
- L9: safeErrorMessage regex suffix broadened

## must_haves

- `SLACK_CHANNEL_ID` validated with `.regex(/^[CG][A-Z0-9]+$/)` in the Zod schema in `config.ts`
- `createStderrLogger` error method applies `safeErrorMessage` to all msgs before `console.error`
- `PERMISSION_ID_RE` exported from `permission.ts` as `new RegExp(`^${PERMISSION_ID_PATTERN}$`)`
- `server.ts` imports `PERMISSION_ID_RE` from `permission.ts` and uses it directly (no `new RegExp()` inline)
- `server.ts` double casts `as unknown as Record<string, unknown>` replaced with `as Record<string, unknown>` + comment
- `safeErrorMessage` regex changed from `[\w-]+` to `[^\s]+`
- `bun test` passes
- `bunx tsc --noEmit` exits 0

<task id="7-01-01" title="M4: Add SLACK_CHANNEL_ID regex validation to Zod schema">

**File:** `src/config.ts`

**Change:** In `ConfigSchema`, replace the current `SLACK_CHANNEL_ID` definition:

```ts
SLACK_CHANNEL_ID: z.string().min(1, 'SLACK_CHANNEL_ID is required'),
```

With:

```ts
SLACK_CHANNEL_ID: z
  .string()
  .min(1, 'SLACK_CHANNEL_ID is required')
  .regex(
    /^[CG][A-Z0-9]+$/,
    'SLACK_CHANNEL_ID must be a Slack channel/group ID (e.g. C0XXX or G0XXX)',
  ),
```

**Note:** The existing `VALID_ENV` in `config.test.ts` uses `'C0123456789'` which passes this regex.

<task id="7-01-04" title="M6: Export PERMISSION_ID_RE from permission.ts and import in server.ts">

**Part 1 — `src/permission.ts`:**

Add a new exported constant after the existing `PERMISSION_ID_PATTERN` export:

```ts
export const PERMISSION_ID_PATTERN = '[a-km-z]{5}'

// Pre-built anchored regex for validating a single permission request ID.
// Exported so callers (server.ts) can use the constant directly instead of
// constructing new RegExp at call time.
export const PERMISSION_ID_RE = new RegExp(`^${PERMISSION_ID_PATTERN}$`)
```

**Part 2 — `src/server.ts`:**

Update the import from `permission.ts` to include `PERMISSION_ID_RE`:

```ts
import {
  formatPermissionRequest,
  PERMISSION_ID_RE,
  parsePermissionReply,
} from './permission.ts'
```

Remove `PERMISSION_ID_PATTERN` from the import (it's no longer used in server.ts).

In the `PermissionRequestSchema` definition, replace:

```ts
request_id: z.string().regex(new RegExp(`^${PERMISSION_ID_PATTERN}$`)),
```

With:

```ts
request_id: z.string().regex(PERMISSION_ID_RE),
```
