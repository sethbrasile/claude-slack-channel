GREEN phase — implement `src/slack-client.ts`:

**Exports needed:**
- `MessageFilter` interface: `{ channelId: string; allowedUserIds: string[] }`
- `SlackEvent` interface: `{ channel?: string; user?: string; bot_id?: string; subtype?: string }`
- `SlackMessage` interface: `{ text: string; user: string; channel: string; ts: string; thread_ts?: string }`
- `MessageHandler` type: `(message: SlackMessage) => void`
- `shouldProcessMessage(event: SlackEvent, filter: MessageFilter): boolean` — pure function
- `isDuplicate(ts: string, seen: Set<string>): boolean` — pure function, injectable seen set
- `createStderrLogger(): Logger` — returns Logger whose methods use console.error
- `createSlackClient(config: ChannelConfig, server: Server): { start(): Promise<void>; stop(): Promise<void> }`

**shouldProcessMessage implementation:**
1. if event.subtype → return false
2. if event.bot_id → return false (catches bot loop even without subtype — Bolt SDK gap)
3. if !event.channel || !event.user → return false
4. if event.channel !== filter.channelId → return false
5. if !filter.allowedUserIds.includes(event.user) → return false
6. return true

**isDuplicate implementation:** Takes `ts` and `seen: Set<string>` parameter. Returns true if seen.has(ts). Otherwise seen.add(ts) and return false. (TTL is handled in createSlackClient via module-level Map — the pure function stays simple for testing.)

**createStderrLogger:** Returns object implementing Logger interface with debug/info/warn/error all using console.error, setLevel: () => {}, setName: () => {}, getLevel: () => LogLevel.INFO.

**createSlackClient:**
- Create module-level `seenTs = new Map<string, number>()` for TTL dedup
- Create `WebClient` with logger: createStderrLogger()
- Create `SocketModeClient` with appToken, logger: createStderrLogger(), autoReconnectEnabled: true
- Register `socketMode.on('slack_event', async ({ event, ack }) => { ... })` handler:
  1. try { await ack() } catch (err) { console.error('ack failed:', err); return }
  2. if event.type !== 'message' return
  3. if event.subtype return
  4. if event.bot_id return
  5. if (!shouldProcessMessage(event, { channelId: config.channelId, allowedUserIds: config.allowedUserIds })) return
  6. TTL dedup: expire old seenTs entries, check/add current ts with 30s TTL
  7. Phase 1 stub: console.error('[slack-client] message received (forwarding not yet implemented)', event.ts)
- Return `{ start: () => socketMode.start(), stop: () => socketMode.disconnect() }`

All Slack API calls in the reply tool (Phase 2) must include `unfurl_links: false, unfurl_media: false` — add a comment noting this requirement in the WebClient usage area.
