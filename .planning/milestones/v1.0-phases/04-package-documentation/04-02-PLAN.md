Walk through the complete operator setup for a single Claude Code project:

1. Create Slack app — direct them to api.slack.com/apps, click "Create New App" > "From a manifest", paste contents of slack-app-manifest.yaml from this repo, create the app

4. Find your channel ID — Right-click the Slack channel > Copy link, extract the C0XXXXXXXXX ID from the URL
5. Invite the bot to the channel — `/invite @Claude` in the target channel

7. Start Claude Code — `claude --dangerously-load-development-channels server:slack`

Title: "Multi-Project VM: Running Multiple Instances"

CRITICAL: This documents MULTIPLE INDEPENDENT SERVER PROCESSES, each with its own channel — NOT a single server handling multiple channels.

Intro: When running Claude Code for multiple projects on a shared machine (e.g., a VM), each project needs its own MCP server instance with its own dedicated Slack channel.

Show:
1. Create one Slack channel per project (e.g., #project-alpha, #project-beta)
2. Create separate .mcp.json per project directory, each with a different SLACK_CHANNEL_ID and SERVER_NAME
3. Show example project structure:
   ```
   ~/projects/
   ├── project-alpha/
   │   └── .mcp.json  # SLACK_CHANNEL_ID=C0ALPHA..., SERVER_NAME=project-alpha
   └── project-beta/
       └── .mcp.json  # SLACK_CHANNEL_ID=C0BETA...,  SERVER_NAME=project-beta
   ```
4. Each project runs its own independent `bunx claude-slack-channel` process
5. ALLOWED_USER_IDS can be the same across all instances (same operators)
6. Why this works: each server instance binds to exactly one channel, so commands from #project-alpha only reach the project-alpha Claude session

Add a note: "Each server instance is completely independent — there is no multi-channel mode in a single server process."
